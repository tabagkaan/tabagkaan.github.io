XOX-UI

Overview: A simple user interface for tic-tac-toe made in Swift. Created to practice designing layouts without gameplay logic.

Key Idea: Focus only on building the UI.

How it Works:

  • The game board is organized inside a ZStack, so squares and text elements can overlap naturally.
  • Each cell is drawn as a blue square using Rectangle().frame(width:120, height:120).
  • X and O are displayed as text views layered above the corresponding square.
  • The nine cells are arranged with .offset(...) values of −140, 0, or 140 for x and y, creating a centered 3×3 grid.

Limitations:

  • No gameplay features.
  • Only static design, no interaction.

XOX-Scratch

Overview: A tic-tac-toe game created with Scratch to learn block-based programming. Fully playable for two players.

Key Idea: Use Scratch blocks to bring a classic game to life.

How it Works:

  • At the start (green flag), initialize the square variable to 0.
  • When a circle is clicked: if its value is 0 or 2, change it to 1; otherwise change it to 2.
  • The X sprite continuously checks the variable: it shows if the value = 1, otherwise it stays hidden.
  • The O sprite continuously checks the variable: it shows if the value = 2, otherwise it stays hidden.
  • This setup is applied consistently across all nine cells of the grid.
  • Additional logic determines when a player has won

Limitations:

  • Simple visuals.
  • No draw detection.
